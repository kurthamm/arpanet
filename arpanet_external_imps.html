Connect Your IMP & PiDP Replicas

A practical path for attaching external replica hardware to this hosted ARPANET implementation.

What Works Today

This page describes Kurt Hamm's hosted deployment and companion PiDP-10 integration. It is not official upstream ARPANET project documentation.

This hosted implementation can accept an external simulated IMP over a stable network link. That external IMP can then attach to a local host simulator, including a PiDP-10 running ITS with NCP enabled.

The tested example in this deployment is a physical PiDP-10 replica attached as ARPANET host 41, shown by NCP as octal host 051. The ARPANET path is verified with the IMP62-to-IMP41 link and ncp-ping 41.

The browser terminal path is separate: this site's @L 41 command attaches to the PiDP-10 SIMH MTY terminal line over Tailscale. That keeps the public terminal usable even when historical NCP TELNET is unreliable.

This is not a promise that every replica machine is plug-and-play. It is a working pattern that can be reproduced and adapted.

Architecture

Browser terminal path: this site's hosted terminal -> WebSocket relay on the DigitalOcean droplet -> PiDP-10 SIMH MTY terminal over Tailscale -> physical PiDP-10 replica running ITS ARPANET validation path: Kurt's ARPANET simulation host -> simulated IMP mesh -> IMP62 site-local remote link -> Pi IMP41 over Tailscale UDP -> PiDP-10 NCP host 41 / octal 051

Minimum Ingredients

  • A local IMP simulator or compatible bridge.
  • A stable UDP path to the ARPANET simulation host.
  • A host simulator or replica with matching network support.
  • A unique ARPANET host number and IMP attachment point.
  • Layered validation: IMP link, NCP ping, and a separate browser terminal path.

Tested PiDP-10 Pattern

  • Pi-side H316 IMP configured as IMP 41.
  • PiDP-10 KA10 simulator running a separate ARPA51 ITS profile.
  • NCP-capable KA10 simulator rebuilt with PiDP front-panel support.
  • Tailscale or equivalent overlay network for stable UDP routing.
  • Droplet-side IMP62 link loaded from site-local runtime configuration.
  • This site's hosted terminal command @L 41 attaches to the PiDP SIMH MTY line, not historical NCP TELNET.

Support and Coordination

External node attachment affects this deployment's shared topology, host numbering, and remote IMP links. Coordinate with Kurt before expecting a permanent public link.

  • Use GitHub issues or repository discussions for reproducible setup questions.
  • Use the PiDP-10 Google Group for replica-specific hardware and software discussion.
  • Keep private IPs, tunnel credentials, disk packs, and passwords out of public posts.
